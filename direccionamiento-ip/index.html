¿Qué es una dirección IP?

Una dirección IP es un identificador único de 32 bits (IPv4) que identifica cada dispositivo en una red TCP/IP.

Formato binario:

11000000.10101000.00000001.00000001

Formato decimal:

192.168.1.1
  • Cada octeto: 0-255 (8 bits)
  • Total de direcciones IPv4: 2³² = 4,294,967,296
  • Formato: Red.Red.Red.Host o Red.Host dependiendo de la clase

Componentes de una dirección IP

Parte de Red (Network)

  • Identifica la red específica
  • Debe ser única en Internet
  • Utilizada por routers para ruteo

Parte de Host

  • Identifica el dispositivo en la red
  • Debe ser única dentro de la red
  • Asignada por administrador local

Ejemplo: 192.168.1.100/24

  • Red: 192.168.1.0 (primeros 24 bits)
  • Host: 100 (últimos 8 bits)
  • Máscara: 255.255.255.0

Máscaras de Subred

La máscara de subred determina qué parte de la dirección IP corresponde a la red y qué parte al host.

Notación CIDR vs Decimal

Notación CIDR:

  • /8 = 255.0.0.0
  • /16 = 255.255.0.0
  • /24 = 255.255.255.0
  • /30 = 255.255.255.252

Hosts disponibles:

  • /24 = 254 hosts
  • /25 = 126 hosts
  • /26 = 62 hosts
  • /30 = 2 hosts

Subnetting Básico

Ejemplo práctico: Dividir 192.168.1.0/24

Red original: 192.168.1.0/24 (254 hosts)
Necesitamos 4 subredes de ~60 hosts cada una

Solución: Usar /26 (62 hosts por subred)

Subred 1: 192.168.1.0/26 (192.168.1.1-62)
Subred 2: 192.168.1.64/26 (192.168.1.65-126)
Subred 3: 192.168.1.128/26 (192.168.1.129-190)
Subred 4: 192.168.1.192/26 (192.168.1.193-254)

Fórmula clave: Número de subredes = 2ⁿ (donde n = bits prestados)

Hosts por subred: 2ᵐ - 2 (donde m = bits de host restantes)

VLSM - Variable Length Subnet Masking

VLSM permite usar diferentes máscaras de subred dentro de la misma red principal, optimizando el uso de direcciones IP.

Ejemplo práctico:

Red: 192.168.100.0/24
Necesidades:
- Red A: 100 hosts → /25 (126 hosts)
- Red B: 50 hosts → /26 (62 hosts)
- Red C: 20 hosts → /27 (30 hosts)
- Enlaces WAN: 2 hosts → /30 (2 hosts)

Asignación VLSM:
192.168.100.0/25 → Red A (0-127)
192.168.100.128/26 → Red B (128-191)
192.168.100.192/27 → Red C (192-223)
192.168.100.224/30 → WAN 1 (224-227)
192.168.100.228/30 → WAN 2 (228-231)

Introducción a IPv6

¿Por qué IPv6?

  • Agotamiento de IPv4
  • Crecimiento de Internet
  • IoT y dispositivos móviles
  • Simplificación del protocolo

Características IPv6

  • 128 bits de longitud
  • 3.4 × 10³⁸ direcciones
  • Autoconfiguración
  • Seguridad integrada (IPSec)
Ejemplo IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
Comprimida: 2001:db8:85a3::8a2e:370:7334

Estructura: 8 grupos de 4 dígitos hexadecimales separados por ":"

NAT - Network Address Translation

NAT permite que múltiples dispositivos con direcciones privadas compartan una dirección IP pública para acceder a Internet.

Tipos de NAT:

  • Static NAT: 1:1 mapping
  • Dynamic NAT: Pool de IPs
  • PAT/NAPT: Overload

Beneficios:

  • Conservación de IPs públicas
  • Seguridad adicional
  • Flexibilidad de red interna
Tabla NAT ejemplo:
Private IP | Public IP | Port
192.168.1.10:1024 → 200.1.1.1:5000
192.168.1.20:1025 → 200.1.1.1:5001
192.168.1.30:1026 → 200.1.1.1:5002

¿Cómo funciona NAT?

Proceso paso a paso:

1. Host interno (192.168.1.10) envía paquete a Internet
Origen: 192.168.1.10:1024 → Destino: 8.8.8.8:53

2. Router NAT traduce la dirección origen
Origen: 200.1.1.1:5000 → Destino: 8.8.8.8:53
[Guarda en tabla: 192.168.1.10:1024 ↔ 200.1.1.1:5000]

3. Respuesta de Internet llega al router
Origen: 8.8.8.8:53 → Destino: 200.1.1.1:5000

4. Router consulta tabla y reenvía al host interno
Origen: 8.8.8.8:53 → Destino: 192.168.1.10:1024

Clave: NAT mantiene una tabla de traducciones para mapear conexiones internas con externas.

Fundamentos de Firewalls

Un firewall es un sistema de seguridad que controla el tráfico de red basándose en reglas predefinidas.

Tipos de Firewalls:

  • Packet Filtering: Inspección básica
  • Stateful: Rastreo de conexiones
  • Application Layer: Inspección profunda
  • Next-Gen: Funciones avanzadas

Criterios de filtrado:

  • Direcciones IP origen/destino
  • Puertos origen/destino
  • Protocolo (TCP/UDP/ICMP)
  • Estado de conexión
