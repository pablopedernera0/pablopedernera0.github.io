iptables: Firewall en Linux

Gestión de tráfico de red y seguridad

¿Qué es iptables?

  • Herramienta de firewalling: Controla el tráfico de red en Linux
  • Acceso al kernel: Utiliza las reglas de Netfilter del kernel
  • Control granular: Filtra paquetes por puertos, protocolos, IPs
  • Estándar de facto: Herramienta clásica en Linux desde 1998
  • Línea de comandos: Configuración flexible y potente

Arquitectura: Tablas y Cadenas

Tablas Principales

  • filter: Filtra paquetes (por defecto)
  • nat: Traduce direcciones (NAT)
  • mangle: Modifica parámetros
  • raw: Procesa antes de tracking

Cadenas (Chains)

  • INPUT: Tráfico entrante
  • OUTPUT: Tráfico saliente
  • FORWARD: Paquetes reenviados
  • Personalizadas: Crear propias

Políticas por Defecto y Acciones

  • ACCEPT: Acepta el paquete
  • DROP: Descarta el paquete (sin respuesta)
  • REJECT: Rechaza y envía respuesta de error
  • LOG: Registra el paquete en log
  • RETURN: Sale de la cadena actual
# Ejemplos de políticas por defecto
iptables -P INPUT DROP
iptables -P OUTPUT ACCEPT
iptables -P FORWARD DROP

Sintaxis Básica de Reglas

iptables -t tabla -A cadena -p protocolo -s origen -d destino --dport puerto -j acción
  • -t: Tabla (filter, nat, mangle)
  • -A: Añadir regla; -I: Insertar; -D: Eliminar
  • -p: Protocolo (tcp, udp, icmp)
  • -s/-d: IP origen/destino
  • --dport: Puerto destino
  • -j: Jump (acción a ejecutar)

Ejemplos Prácticos de Reglas

# Permitir SSH
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Permitir HTTP y HTTPS
iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT

# Bloquear IP específica
iptables -A INPUT -s 192.168.1.100 -j DROP

# Permitir loopback
iptables -A INPUT -i lo -j ACCEPT

NAT y Port Forwarding

  • NAT: Traducción de direcciones de red
  • SNAT: Traduce dirección origen
  • DNAT: Traduce dirección destino
  • Masquerading: Oculta IPs internas
# Port Forwarding: puerto 8080 al 80
iptables -t nat -A PREROUTING -p tcp --dport 8080 -j DNAT --to-destination :80

# Masquerading (NAT saliente)
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

Visualizar y Gestionar Reglas

# Listar reglas
iptables -L -n -v
iptables -L -t nat

# Limpiar todas las reglas
iptables -F
iptables -t nat -F

# Ver estadísticas
iptables -L -v -x

# Guardar configuración
iptables-save > /etc/iptables/rules.v4

Persistencia de Reglas

  • Problema: Las reglas se pierden al reiniciar
  • iptables-persistent: Guardar reglas automáticamente
  • iptables-save: Exportar configuración a archivo
  • iptables-restore: Restaurar desde archivo
# En Debian/Ubuntu
apt-get install iptables-persistent
netfilter-persistent save

# En RHEL/CentOS
systemctl enable iptables
service iptables save

nftables: La Evolución

  • Sucesor moderno: Reemplaza iptables, ip6tables, arptables
  • Sintaxis mejorada: Más clara y consistente
  • Mejor rendimiento: Menos código kernel
  • Menos dependencias: Una herramienta unificada
  • Compatibilidad: Aún se usa iptables en muchos sistemas
# Equivalente en nftables
nft add rule inet filter input tcp dport 22 accept
nft add rule inet filter input drop
